# Internationalization (i18n)

Angular provides built-in i18n support for translating templates, handling pluralization, and managing localized builds.

## Setup

```bash
ng add @angular/localize
```

## Mark Text for Translation

```html
<h1 i18n>Welcome to the dashboard</h1>
<button i18n-title title="Create">New</button>
```

## ICU Expressions

```html
<p i18n>
  {messages, plural, =0 {No messages} =1 {One message} other {# messages}}
</p>
```

## Extract Messages

```bash
ng extract-i18n --format=xlf
```

This generates `messages.xlf` in your project root.

## Build Localized Versions

Configure locales in `angular.json`, then build:

```bash
ng build --localize
```

## Best Practices

- Use descriptive meaning and description metadata for translators
- Keep text in templates rather than concatenated strings
- Validate right-to-left layouts early
